🔥 Firebase 설정 가이드

예약 시스템에 Firebase 데이터베이스를 연결하는 방법을 안내합니다

1Firebase 프로젝트 생성

Firebase 콘솔에서 새 프로젝트를 생성합니다.

  1. Firebase 콘솔 열기
  2. "프로젝트 추가" 또는 "Add project" 클릭
  3. 프로젝트 이름 입력 (예: "restaurant-reservation")
  4. Google Analytics 활성화 (선택사항)
  5. 프로젝트 생성 완료

2Firestore 데이터베이스 설정

Firestore 데이터베이스를 활성화하고 보안 규칙을 설정합니다.

  1. 생성된 프로젝트에서 "Firestore Database" 클릭
  2. "데이터베이스 만들기" 클릭
  3. 보안 규칙 모드 선택:
    개발용: 테스트 모드 (모든 읽기/쓰기 허용)
    운영용: 프로덕션 모드 (보안 규칙 설정 필요)
  4. 위치 선택 (asia-northeast3 - 서울 권장)
  5. 데이터베이스 생성 완료

3웹 앱 설정

웹 애플리케이션을 위한 Firebase 설정을 가져옵니다.

  1. 프로젝트 개요에서 웹 아이콘() 클릭
  2. 앱 닉네임 입력 (예: "restaurant-web")
  3. "Firebase Hosting 설정" 체크 해제
  4. "앱 등록" 클릭
  5. Firebase SDK 설정 코드 복사
설정 코드 예시:
const firebaseConfig = { apiKey: "AIzaSyB...", authDomain: "your-project.firebaseapp.com", projectId: "your-project-id", storageBucket: "your-project.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abcdef..." };

4설정값 적용

복사한 설정값을 프로젝트에 적용합니다.

설정값 입력

생성된 설정 코드

다음 단계: 생성된 코드를 복사하여 firebase-config.js 파일의 firebaseConfig 객체를 교체하세요.

5보안 규칙 설정 (선택사항)

운영 환경에서는 적절한 보안 규칙을 설정해야 합니다.

// Firestore 보안 규칙 예시 rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // 예약 데이터는 모든 사용자가 읽기/쓰기 가능 (개발용) match /reservations/{document} { allow read, write: if true; } // 운영용 보안 규칙 (관리자만 접근) // match /reservations/{document} { // allow read, write: if request.auth != null && // request.auth.token.admin == true; // } } }

6테스트 및 확인

설정이 올바르게 적용되었는지 확인합니다.

  1. 데이터베이스 테스트 페이지로 이동
  2. "Firebase 연결 테스트" 버튼 클릭
  3. ✅ Firebase 실제 설정이 감지되었습니다 메시지 확인
  4. "데이터 작업 테스트" 버튼으로 실제 저장/조회 테스트
⚠️ 주의사항:
  • API 키는 공개되어도 되지만, 보안 규칙으로 접근을 제한하세요
  • 운영 환경에서는 적절한 보안 규칙을 설정하세요
  • Firebase 사용량에 따라 요금이 부과될 수 있습니다
✅ 완료 후:
  • 예약 데이터가 Firebase에 저장됩니다
  • 관리자 페이지에서 실시간으로 데이터를 확인할 수 있습니다
  • 로컬 저장소는 백업으로 계속 작동합니다
